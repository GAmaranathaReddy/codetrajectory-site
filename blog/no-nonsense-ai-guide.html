1. Large Language Models (LLMs)

This diagram shows the flow of user input through prompt engineering, into a large language model (LLM), and finally producing an output. Each box represents a key stage in building and deploying LLM-powered applications.

These are the foundation of nearly every AI product today (think ChatGPT, Claude, Gemini).

  • 1. Prompt design (zero-shot, CoT, role-based)
  • 2. Fine-tuning (LoRA, QLoRA, PEFT)
  • 3. Embeddings for smarter context and search
  • 4. Function calling (integrate with tools/APIs)
  • 5. Hallucination handling (critical for production)

Tools: Ollama, GPT-4o, Claude, Gemini, Hugging Face, Cohere

2. Retrieval-Augmented Generation (RAG)

This diagram illustrates the Retrieval-Augmented Generation (RAG) pipeline, where user queries are processed, relevant data is retrieved from a knowledge base, and combined with LLMs to generate grounded, accurate responses.

Every serious chatbot or assistant uses RAG to fetch real-time knowledge from your own data.

  • 1. Chunking + vector DB indexing
  • 2. Smart search pipelines
  • 3. Dynamic context injection
  • 4. Multi-source retrieval (APIs, files, web)
  • 5. Grounded prompting

Tools: FAISS, Pinecone, LangChain, Weaviate, ChromaDB, Haystack

3. Agentic AI & AI Agents

This diagram visualizes agentic AI systems, showing how multiple agents (planner, executor, memory) interact and coordinate to solve complex tasks using tools and feedback loops.

The next evolution is multiple agents coordinating on tasks (research, planning, workflows).

  • 1. Planner-executor designs
  • 2. Long-term memory support
  • 3. Multi-agent messaging
  • 4. Feedback loops for self-improvement
  • 5. Tool/API orchestration

Tools: CrewAI, LangGraph, AgentOps, FlowiseAI, Superagent, ReAct

4. AI Engineer (Deployment & Ops)

This diagram outlines the deployment and operations pipeline for AI systems, from source code and CI/CD to model deployment, API endpoints, and monitoring.

This is about shipping AI systems, not just tinkering with notebooks.

  • 1. AI workflows orchestration
  • 2. Model deployment & version control
  • 3. API security & CI/CD pipelines
  • 4. Latency & cost optimizations
  • 5. Responsible AI practices

Tools: Docker, FastAPI, Hugging Face Hub, LangSmith, Cloudflare Workers, GitHub Copilot

5. Machine Learning Engineer

This diagram shows the machine learning pipeline, including data cleaning, feature engineering, model training, evaluation, and deployment for scalable ML solutions.

Still important! These skills help build foundational models and pipelines.

  • 1. Data cleaning & feature engineering
  • 2. Classical ML (XGBoost, SVM)
  • 3. Deep learning (TensorFlow, PyTorch)
  • 4. Model evaluation and tuning
  • 5. MLOps and cloud scaling

Tools: Scikit-learn, TensorFlow, PyTorch, MLflow, Vertex AI, Kubeflow, Airflow, DVC
